# SLAY THE MONSTARZ — 아트 디렉션 (v2)

- 작성일: 2026-06-27
- 목적: **모든 이미지·애니메이션 생성의 단일 기준.** 생성 프롬프트는 이 문서를 따른다. v1처럼 무국적 AI 결과물로 새는 걸 막는다.
- 목표: Slay the Spire 톤 90% + MONSTARZ 정체성.

## 1. 한 줄 정의

**"치비 형태를, 슬더스식 손그림 다크판타지 톤으로 그린 MONSTARZ 전사."**

> **승인된 스타일 앵커 (2026-06-27)**: `assets/game/jdd_battle.png` (원본 `assets/masters/jdd/jdd_master.png`). 지두두(jdd, 여성, T) — 치비 + 슬더스 손그림 2D + 테란 녹슨 강철 전투복 + 등번호 단서 + 측면 우향. **이후 모든 캐릭터는 이 톤·완성도에 맞춘다.**

## 2. 확정 사항 (사용자 결정)

| 항목 | 결정 |
|---|---|
| **비율** | 치비 유지 (큰 머리, 작은 몸) |
| **렌더/채색** | 슬더스식 손그림 2D — 페인터리, 두꺼운 어두운 외곽선, 어둡고 차분한 다크판타지 팔레트, 약간 거친 질감, 극적인 명암 |
| **의상** | 종족(P/T/Z) 테마 전투복으로 재설계. **야구 저지 그대로 입히지 않음.** 멤버 단서(얼굴·헤어·등번호/팀 엠블럼)를 전투복에 녹여 정체성 유지 |
| **방향(전투)** | 측면/3-4 뷰. 플레이어=오른쪽(적) 바라봄, 적=왼쪽(플레이어) 바라봄. **정면 금지** |
| **방향(캐릭터/로스터 카드)** | 정면 (기존 FUT 캐릭터 카드용) |
| **전투 능력 카드 아트** | 공용 카드는 `cardId`별 1장. 인물 초상이 아니라 무기·충격·방어·에너지 등 능력 모티프 중심. 캐릭터 식별 아트는 시그니처 카드만 예외 |
| **정체성 고정 범위** | 레퍼런스(`assets/refs/`)에서 **얼굴·헤어만** 고정. 몸·의상·자세·무기·배경은 새로 설계 |

## 3. 슬더스 톤 체크리스트 (90% 기준)

- 페인터리 손그림 2D (3D 글로시 렌더 아님)
- 두껍고 어두운 외곽선 + 내부 음영
- 어둡고 채도 낮은 다크판타지 팔레트, 포인트 색만 채도 높게
- 강한 실루엣 (작은 전투 크기에서도 즉시 구분)
- 드라마틱한 측광/역광
- 배경과 분리되는 또렷한 가장자리 (색 번짐 없음)

## 4. 종족 시각 언어 (치비 + 다크판타지로 번안)

- **P (프로토스)**: 매끈한 대칭 장갑, 청록·금 에너지 라인, 부유 보호막 파츠 — 단, 글로시 SF 아닌 **고대 유물 같은 손그림 질감**으로.
- **T (테란)**: 산업 강철, 리벳·패널, 배기구, 반동 — **녹슨 중장갑 용병 느낌**으로.
- **Z (저그)**: 비대칭 생체 갑각, 가시, 근육, 산성 — **어두운 괴물스러운 바이오** 톤으로.

## 5. MONSTARZ 정체성 유지법

- **얼굴·헤어**: 레퍼런스 인물로 즉시 인식되게(치비화해도 특징 유지). mini=변현제(P), soulkey=김민철(Z).
- **단서**: 등번호, 팀 엠블럼, 저지 색 배합을 전투복·갑옷 디테일에 삽입(저지를 통째로 입히지 말 것).
- 캐릭터/로스터 카드(정면)에선 얼굴 식별성이 1순위, 전투 스프라이트(측면)에선 실루엣·헤어·색으로 식별.
- 전투 능력의 공용 카드 아트에는 얼굴·헤어·등번호 등 특정 캐릭터 식별 요소를 넣지 않는다. 같은 `cardId`를 쓰는 모든 캐릭터가 같은 이미지를 공유한다.

## 6. 절대 금지 (v1 실패 목록)

- 무국적 매끈한 SF/모바일게임 3D 전사
- 전투 스프라이트 정면 응시
- 캐릭터를 테두리 박스/패널 안에 가두는 레이아웃
- 야구 저지를 그대로 입힌 채 다크판타지라고 우기기
- "파일 나왔다"를 "품질 합격"으로 자체 판정 (품질 판단은 사용자)

## 7. 생성 파이프라인 (도구 분담)

- **Aether 사용 중단 (사용자 방침, 2026-06-27)**: 신규 마스터·모션에는 AetherForge와 make-sprite를 사용하지 않는다. 기존 지두두 라이브 시트는 과거 산출물로만 유지한다.
- **마스터 정적 원화**: 승인된 레퍼런스를 넣은 `GPT Image` 또는 `Nano Banana / Nano Banana Pro` 직접 생성. 프롬프트는 본 문서 2~6절을 반영한다.
- **동작·애니메이션 기본 단위**: 캐릭터마다 개별 프레임 수동 생성 대신 **소수 키포즈 보드**를 만든다.
  - 일반 적·작은 실루엣: 3×2 보드 1장(대기 2, 공격 3, 피격 1).
  - 대형 보스: 해상도 보존이 필요하면 3포즈 보드 2장으로 분리한다.
  - 보드 생성 → `tools/keypose_board.py`로 크로마 제거·셀 분리·알파 경계 QA → 엔진에서 트윈/크로스페이드로 연결한다.
  - 셀 안전 여백을 먼저 자동 검사하고, 실패한 셀이나 시그니처 포즈만 승인 원화를 ref로 개별 재생성한다.
  - 머즐 섬광·투사체·연기·임팩트는 스프라이트에서 빼고 Pixi VFX로 처리한다.
- 후보는 `assets/review/`에 보존하고 사용자 승인분만 `assets/game/`으로 승격한다.
- 투명 배경: `#00ff00` 크로마키 → 로컬 제거 + 셀별 알파블리드.

## 8. 검수 기준 (캐릭터 생성물 합격 조건)

- [ ] 슬더스 손그림 2D 톤인가 (3D 글로시 아님)
- [ ] 치비 비율인가
- [ ] 레퍼런스 인물로 인식되는가
- [ ] 종족(P/T/Z) 테마가 읽히는가
- [ ] 전투용은 측면 지향인가
- [ ] 멤버 단서(번호/엠블럼/색)가 있는가
- [ ] 작은 전투 크기에서 실루엣 구분되는가
- [ ] 가장자리 색 번짐 없는가

> 위 8개 중 하나라도 미달이면 승격 금지. 최종 판단은 사용자.

공용 능력 카드 아트는 인물 식별 항목 대신 `cardId`의 행동 의미가 작은 카드 크기에서도 즉시 읽히는지 검수한다.
